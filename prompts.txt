New Game Handler (lines 134-138)

Bugs Found
1. status is never reset (Critical)
The handler doesn't reset st.session_state.status. After winning or losing, status stays "won" or "lost". On rerun, line 140 immediately calls st.stop(), making it impossible to play a new game after any completed game.

Fix: Add st.session_state.status = "playing" to the handler.

2. history is never cleared
st.session_state.history accumulates guesses across games. Guesses from a previous game pollute the debug panel of the new game.

Fix: Add st.session_state.history = [] to the handler.

3. attempts starts at 0 instead of 1
Initial state sets attempts = 1 (line 96), but the New Game handler sets it to 0. The first submit increments to 1, so behavior is identical — but it's an inconsistency that makes the attempt count display show one extra attempt on game start.

4. Hardcoded range ignores difficulty
Line 136 always uses random.randint(1, 100) regardless of the difficulty setting. Selecting Easy (range 1–20) or Hard (range 1–50) and clicking New Game silently generates a number outside the intended range.

Fix: Use random.randint(low, high) where low, high come from get_range_for_difficulty(difficulty).

New game on Easy/Hard difficulty	Secret in difficulty range	Always 1–100
New game multiple times	Each gives new secret	Works correctly

# Plan: Fix New Game Button Bugs in app.py

## Context
The New Game button in the Glitchy Guesser app has several bugs discovered during edge case analysis. Clicking New Game after winning/losing leaves the game in a broken state (can't play again), old history persists across games, attempt count starts at the wrong value, and the difficulty range is ignored when generating the new secret.

## Critical File
- `/Users/ananyak/Desktop/Codepath-AI110/ai110-module1show-gameglitchinvestigator-starter/app.py`

---

## Changes

### 1. Fix `new_game` handler (lines 134-138)

**Current:**
```python
if new_game:
    st.session_state.attempts = 0
    st.session_state.secret = random.randint(1, 100)
    st.success("New game started.")
    st.rerun()
```

**Fixed:**
```python
if new_game:
    st.session_state.attempts = 1
    st.session_state.secret = random.randint(low, high)
    st.session_state.status = "playing"
    st.session_state.history = []
    st.success("New game started.")
    st.rerun()
```

Changes:
- `attempts = 1` (consistent with initial state on line 96)
- `random.randint(low, high)` (uses difficulty-based range instead of hardcoded 1–100)
- `status = "playing"` (allows play after win/loss — critical bug fix)
- `history = []` (clears old guesses)

---

### 2. Fix hardcoded range string in `st.info` (line 110)

**Current:**
```python
st.info(
    f"Guess a number between 1 and 100. "
    f"Attempts left: {attempt_limit - st.session_state.attempts}"
)
```

**Fixed:**
```python
st.info(
    f"Guess a number between {low} and {high}. "
    f"Attempts left: {attempt_limit - st.session_state.attempts}"
)
```
